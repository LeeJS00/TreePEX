### H. Compact analytic priors (2) — Sakurai-Tamaru 식

| Feature | Unit | 공식 |
|---|---|---|
| `compact_gnd_estimate_fF` | fF | Σ_i ε₀·ε_r,i · A_i / d_i, d_i = max(0.05, layer_i × 0.1) μm |
| `compact_cpl_estimate_total_fF` | fF | Σ_edge ε₀·(ε_r_l1 + ε_r_l2)/2 · (broadside + lateral)/d, d = max(0.05, surface_dist_um) |

**V3 합계 = 3 + 3 + 10 + 12 + 3 + 3 + 5 + 2 = 41 ✓**

(참고: 코드는 `fanout` = `n_aggressor_nets` 동일값을 별도로 emit하지만 redundant; XGBoost 학습 시 모델이 중복 split을 흡수.)

## V4 H3 features (26-D) — tile pkl.gz cache에서 추출

V4는 `/data/PINNPEX/data/processed_v3/intel22/<design>/*.pkl.gz` tile cache 사용 (14 × 14 μm xy window 단위로 cuboid 그룹화).

### A. Self check (1)

| Feature | 계산 |
|---|---|
| `target_n_cuboids_check` | V4가 본 target cuboid 개수 (max V4 subsample = 256) |

### B. Aggressor distribution counts (5)

| Feature | 계산 |
|---|---|
| `agg_n_distinct` | tile 내 distinct aggressor net 수 |
| `agg_count_above_target_z` | target 평균 z보다 높은 aggressor cuboid 수 |
| `agg_count_below_target_z` | target 평균 z보다 낮은 aggressor cuboid 수 |
| `agg_count_within_1um_xyz` | target 평균과 3D 거리 ≤ 1 μm aggressor cuboid 수 |
| `agg_count_within_3um_xyz` | ≤ 3 μm |
| `agg_count_within_5um_xyz` | ≤ 5 μm |

### C. Top-3 aggressor pair geometry (18 = 6 × 3)

각 aggressor `a`마다 score를 매기고 (overlap × decay), top-3 aggressor를 선택. K ∈ {1, 2, 3}:

| Feature | Unit | 계산 |
|---|---|---|
| `topK_score` | – | overlap_area × exp(−min_xy_dist / 1 μm) |
| `topK_overlap_um2` | μm² | target과 그 aggressor의 합산 xy overlap |
| `topK_min_xy_dist_um` | μm | 페어 사이 최소 xy bbox-edge distance |
| `topK_mean_dz_um` | μm | 페어 사이 평균 z 차이 |
| `topK_agg_size_um2` | μm² | 그 aggressor의 cuboid total metal area |
| `topK_layer_diff_flag` | 0/1 | aggressor 평균 z와 target 평균 z 차이가 0.3 μm 초과면 1 |

### D. Concentration (1)

| Feature | 계산 |
|---|---|
| `topk_score_concentration` | sum(topK score, K=1..3) / sum(all aggressor scores) |

**V4 합계 = 1 + 5 + 18 + 1 + 1 = 26 ✓**

## 두 그룹의 의미적 분업

| 그룹 | 의미 | 잡는 물리 |
|---|---|---|
| **V3 41-D** | net 전체에 대한 *aggregate* 통계 | 평행판 ground cap (compact_gnd), 평균 coupling 환경, layer stack |
| **V4 26-D** | net의 *지역적 hotspot* (top aggressor 3) | 1-3개의 dominant aggressor가 coupling의 대부분을 결정 |

V3는 *background environment*, V4는 *foreground hotspot*. Tree 모델은 두 view를 자동 결합:
- 어떤 net은 V3-driven (배경 dominant)
- 어떤 net은 V4-driven (강한 1:1 coupling dominant)

ablation: V3-only 학습 시 천장 4.66 % (4-way oracle bound). V4 26-D 추가 → 4.98 %.
